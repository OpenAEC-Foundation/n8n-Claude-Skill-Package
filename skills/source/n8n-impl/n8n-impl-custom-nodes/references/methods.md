# Custom Node Development: Methods Reference

## Project Structure (Complete)

```
n8n-nodes-<service-name>/
├── package.json              # CRITICAL: n8n field registers nodes/credentials
├── tsconfig.json             # TypeScript configuration (extends n8n base)
├── .eslintrc.js              # Linting config (use n8n preset)
├── credentials/
│   ├── MyApi.credentials.ts  # API key / OAuth credential
│   └── MyOAuth2.credentials.ts  # OAuth2 credential (if needed)
├── nodes/
│   └── MyNode/
│       ├── MyNode.node.ts    # Main node implementation
│       ├── MyNode.node.json  # Codex metadata (search/AI categorization)
│       └── resources/        # Resource/operation descriptions (declarative)
│           ├── user/
│           │   ├── index.ts
│           │   ├── create.ts
│           │   └── getAll.ts
│           └── post/
│               ├── index.ts
│               └── get.ts
├── icons/
│   └── my-icon.svg           # Node icon (SVG preferred, PNG accepted)
├── dist/                     # Compiled output (generated by npm run build)
├── node_modules/
├── .gitignore
└── README.md                 # Package documentation
```

### n8n Field Breakdown

| Property | Purpose | Rule |
|----------|---------|------|
| `n8nNodesApiVersion` | API compatibility version | ALWAYS set to `1` |
| `strict` | Enable strict validation | ALWAYS set to `true` |
| `credentials` | Array of credential file paths | ALWAYS point to `dist/*.js` compiled files |
| `nodes` | Array of node file paths | ALWAYS point to `dist/*.js` compiled files |

## Node Registration Format

Each entry in `n8n.nodes` and `n8n.credentials` MUST follow this format:

```
dist/<folder>/<FileName>.<type>.js
```

Where `<type>` is:
- `.node.js` for node files
- `.credentials.js` for credential files

### Multiple Nodes Example

```json
{
```

## Development Commands

### npm run dev

Starts n8n in development mode with your custom node loaded:
- Watches for file changes and auto-reloads
- Accessible at `http://localhost:5678`
- Uses `@n8n/node-cli` under the hood

### npm run build

Compiles TypeScript to JavaScript in `dist/`:
- Runs `n8n-node build` which handles TypeScript compilation
- Copies icon files and JSON metadata to `dist/`
- ALWAYS run before publishing or linking

### npm run lint

Checks code quality using n8n's ESLint rules:
- Validates node naming conventions
- Checks property definitions
- Ensures description completeness

### npm link (Local Testing)

```bash
# Step 1: In your custom node package directory
cd /path/to/n8n-nodes-myservice
npm run build
npm link

# Step 2: Find your n8n installation's node_modules
# For global n8n:
cd $(npm root -g)/n8n
npm link n8n-nodes-myservice

# For local n8n installation:
cd /path/to/n8n
npm link n8n-nodes-myservice

# Step 3: Restart n8n
n8n start
```

ALWAYS run `npm run build` before `npm link` — linking exposes the `dist/` directory.

## Key IExecuteFunctions Methods

| Method | Purpose | Example |
|--------|---------|---------|
| `this.getInputData()` | Get all input items | `const items = this.getInputData()` |
| `this.getNodeParameter(name, i)` | Get parameter value for item i | `this.getNodeParameter('userId', i) as string` |
| `this.getCredentials(name)` | Get decrypted credentials | `await this.getCredentials('myApi')` |
| `this.helpers.httpRequest(opts)` | Make HTTP request | See examples |
| `this.continueOnFail()` | Check if "Continue On Fail" is enabled | Used in catch block |
| `this.getWorkflowStaticData(type)` | Get persistent workflow data | `this.getWorkflowStaticData('node')` |
| `this.helpers.returnJsonArray(data)` | Convert data to INodeExecutionData[] | For trigger nodes |

## Key ICredentialType Properties

| Property | Type | Purpose |
|----------|------|---------|
| `name` | `string` | Internal credential name (must match node reference) |
| `displayName` | `string` | Human-readable name in UI |
| `documentationUrl` | `string` | Link to API docs |
| `properties` | `INodeProperties[]` | Credential fields (API key, URL, etc.) |
| `authenticate` | `IAuthenticate` | Auto-authentication config for declarative nodes |
| `test` | `ICredentialTestRequest` | Credential test configuration |

## IAuthenticate Types

### Generic (Header/Query)

```typescript
authenticate: IAuthenticate = {
    type: 'generic',
    properties: {
        headers: {
            Authorization: '=Bearer {{$credentials.apiKey}}',
        },
    },
};
```

### Generic (Query Parameter)

```typescript
authenticate: IAuthenticate = {
    type: 'generic',
    properties: {
        qs: {
            api_key: '={{$credentials.apiKey}}',
        },
    },
};
```

## Credential Testing

```typescript
export class MyApi implements ICredentialType {
    name = 'myApi';
    displayName = 'My API';
    properties: INodeProperties[] = [/* ... */];

    test: ICredentialTestRequest = {
        request: {
            baseURL: '={{$credentials.baseUrl}}',
            url: '/me',
            headers: {
                Authorization: '=Bearer {{$credentials.apiKey}}',
            },
        },
    };
}
```

The `test` property defines an HTTP request n8n sends when the user clicks "Test Credential" in the UI. A 2xx response means the credential is valid.
